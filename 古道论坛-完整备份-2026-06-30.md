## 四、数据库结构

### 所有表

| 表名 | 用途 | 主要字段 |
|------|------|----------|
| profiles | 用户资料 | id, username, display_name, role, member_no, phone, date_of_birth, gender, hobbies, bio, resume, created_at |

| chat_rooms | 聊天室 | id, name, slug, description, icon, created_at |

## 九、恢复步骤（从零开始）

```bash
# === 1. 环境准备 ===
# 安装 Node.js v24+
# 安装 Git

# === 2. 克隆代码 ===
git clone https://github.com/yuhaohao13-stack/gudao-forum.git
cd gudao-forum
npm install

# === 3. 配置环境变量 ===
# 创建 .env.local，写入：
# NEXT_PUBLIC_SUPABASE_URL=https://rsndnhdimruisysacujg.supabase.co
# NEXT_PUBLIC_SUPABASE_ANON_KEY=sb_publishable_ZcrY62Opev_eOmhWlKpCqg_NZPjy3Z8

# === 4. 初始化数据库 ===
# 打开 https://supabase.com/dashboard/project/rsndnhdimruisysacujg
# SQL Editor → 依次运行 supabase-*.sql（按第八节顺序）

# === 5. 创建 Storage Bucket ===
# Supabase → Storage → New bucket → forum-images（public）

# === 6. 设置管理员 ===
# 先注册一个账号，然后运行：
# UPDATE public.profiles SET role = 'admin' WHERE email = '你的邮箱';

# === 7. 本地开发 ===
npm run dev  # → http://localhost:3000

# === 8. 部署到 Vercel ===
# GitHub 仓库 → Vercel Import → 添加环境变量 → Deploy
# 自定义域名指向 www.gudaoforum.com
```
